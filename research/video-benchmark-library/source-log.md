## Key Observations Per Example

### 1 · Slack
- Opens with single universally recognizable pain (email overload); 25% of runtime on pain before product
- Chaos of scattered shapes resolves into ordered structures — metaphor enacts the product promise
- **Lesson:** One pain + one metaphor beats a feature checklist. Let visual order be the demo.

### 2 · Figma Config 2024
- Motion verbs (Shift / Transform / Amplify) each map to a product promise — motion is semantically load-bearing
- Scalable component library built before production; 15fps chosen deliberately for tactile warmth
- **Lesson:** Define motion as a vocabulary before animating. Over-smoothness signals genericness.

### 3 · Notion (fan explainer)
- Rapid 5–8s use-case vignettes argue breadth without overloading; editing pace itself reads as "flexible"
- Visual language matches product aesthetic (neutral, typographic, white space)
- **Lesson:** When a product does many things, let editorial tempo make the versatility argument.

### 4 · HubSpot
- Live-action grounds pain in human reality; motion graphics handle abstraction — mode switch is scripted
- Second-person narration ("you," "your team") keeps buyer as protagonist
- **Lesson:** Hybrid live + motion works when the mode switch is scripted, not arbitrary.

### 5 · Webflow
- Before-state is made uncomfortable enough that the reveal creates genuine relief
- Hard-cut tonal shift (chaos → sudden order) enacts the product benefit through editing
- **Lesson:** Make the before-state embarrassing. Contrast is the argument.

### 6 · Linear
- Dark mode + near-black signals alignment with engineering workflow (visual empathy)
- Motion confined to micro-interactions; typography carries hierarchy without color dependency
- **Lesson:** For technical audiences, what you don't animate is as important as what you do.

### 7 · CrowdStrike
- Platform modules shown as spatial connected-node diagram — scope comprehensible at a glance
- Threat animation has narrative function: enters → detected → neutralized
- **Lesson:** For multi-module platforms, a spatial diagram is more persuasive than sequential feature walkthroughs.

### 8 · Wiz
- Narrative = information asymmetry problem (you can't see your cloud risk) → product resolves it
- No humor; authoritative + urgent tone matches CISO's emotional state (fear, not delight)
- **Lesson:** In cybersecurity, lead with the threat model. The register is fear-reduction, not delight.

### 9 · Palo Alto
- Three-act structure: old way → risk → new approach
- Color codes threat states: red = threat, blue/green = protected, gray = uncovered
- **Lesson:** Color-coding security posture (threat/covered/exposed) creates instant comprehension.

### 10 · Lacework
- One concept per video; shared visual language makes the series feel like a system
- Concept sequence mirrors buyer journey (awareness → evaluation)
- **Lesson:** Modular short-form series outperforms single omnibus for complex platforms.

### 11 · Stripe
- Single animating metaphor (flowing metallic wave) varies across entire identity — never needs reinvention
- Visual language targets developer/API buyer: code snippets, dark bg, clean UI overlays
- **Lesson:** For existing customers, show-don't-tell works. For acquisition, problem-first still required.

### 12 · Ramp
- Financial workflow abstracted to icon-based flow: receipt → automated approval node
- Green = money/approval; gray = automation — category-standard color grammar
- Quantified outcomes animated as stats, not spoken as claims
- **Lesson:** For workflow automation, animate the transformation itself using the same visual elements before→after.

### 13 · Brex
- Integration visualized as two systems "clicking together" (node connection) — not API calls or schemas
- Finance-professional tone; information density appropriate for CFO audience
- **Lesson:** For integration/extension video, lead with what the buyer can stop doing.

### 14 · Datadog
- Infrastructure concepts → infographic animated diagrams: metrics = bars, traces = arrows, systems = nodes
- Purple + orange color system is internally consistent across all videos — recognizability compounds
- **Lesson:** Build a consistent video color system; repeated recognizability = trust over time.

## Proven 60–90 Second Narrative Structures

| Structure | Best Use | Beat Pattern (75s) |
|---|---|---|
| PAS (Problem → Agitate → Solve) | Most B2B scenarios | 0–12s pain / 12–28s agitate / 28–45s product / 45–65s benefits / 65–70s proof / 70–75s CTA |
| Contrast (Without → With) | Webflow / Slack style | 30–40% before-state; hard-cut to after; motion enacts transformation |
| Architecture First | Multi-module platforms (security, infra) | Diagram first → animate coverage → walk modules |
| Vignette Sequence | Versatile platforms (Notion, HubSpot) | 5–8s scenes in rapid succession; editing pace = versatility argument |
| Update-Led | Existing customer video only | Assumes trust; leads with what's new; never for acquisition |
